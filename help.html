Getting started

Fundamentals of the app and data model

What is Bálggis?

Bálggis is a strategic mapping tool that lets you build and visualise your organisation's structures — from strategic goals to IT systems and everything in between. You can draw connections between objects, define properties, filter for different audiences and export to images or spreadsheets.

The app runs directly in the browser. Data is stored in the browser's local storage (localStorage) — no login, no server, no installation.

Three view modes

  • Map – interactive canvas with nodes and connections
  • Table – structured table view with inline editing
  • Matrix – connection matrix between two categories

Switch between views using the buttons at the top of the toolbar.

Your data is safe

  • Nothing is sent to a server
  • Everything is saved automatically and locally
  • Export a full backup as a JSON file
  • Import JSON to restore your map
⚠️
Clearing browser data will delete the map. Export a JSON backup regularly.

The data model – three layers

1
Categories

Define a type of object (e.g. "KPI", "IT system"). Each category has a colour and a set of parameters.

2
Nodes / Objects

Concrete instances of a category (e.g. "Customer satisfaction" as a KPI). Nodes hold values for all of the category's parameters.

3
Relations / Connections

Links between nodes with a specific type (hierarchy, measures, supports, delivers to).

💡
Demo data – The app starts with a ready-made strategic map the first time you open it. Feel free to explore it before building your own.

Categories

Define the object types in your organisation

What is a category?

A category is a type of object you want to map — for example Strategic goal, KPI, Capability, Initiative or IT system. All objects (nodes) of the same type belong to the same category and share the same parameters (columns).

Categories appear in the left-hand side panel in map view, and as blocks in table view.

Create a new category

1
Click "+ Category"

In the left-hand side panel in map view.

2
Give it a name and choose a colour

The colour is used in the map and table to distinguish categories visually.

3
Add parameters

Click "Edit" on the category to add custom fields.

Edit a category

Click "Edit" on a category to:

  • Change name and colour
  • Add, edit or delete parameters
  • Reorder parameters
  • View and manage the nodes in the category
💡
Parameters can be added and changed at any time — existing nodes retain their values.

Calculated parameters – formula syntax

Use {ParamName} to reference other numeric parameters in the same category. Standard mathematical functions are supported:

// Simple arithmetic
{Revenue} - {Cost}

// Percentage calculation with rounding
round({Actual} / {Target} * 100)

// More complex expressions
round(({Revenue} - {Cost}) / {Revenue} * 100)

Supported functions:

round floor ceil min max abs sqrt pow

Status aggregation

For parameters of type Status (traffic light) you can enable aggregation: "Aggregate status from connected nodes". The status is then computed automatically as the worst status among all nodes pointing into this node.

💡
Example: A KPI with aggregated status will show red if any one of the underlying initiatives has red status.
🔗

Relations / Connections

Define relationships between objects

Relations (edges) bind nodes together and give the map meaning. There are four relation types which can be customised in Settings.

Hierarchy Dark blue, solid line. Used for parent/child relationships, e.g. goal → capability.
Measures Green, dashed line. A KPI measures a strategic goal.
Supports Blue, dashed line. An initiative supports or contributes to a capability.
Delivers to Orange, solid line. An IT system delivers to or enables a capability.

Creating a connection in map view

1
Hover over a node

A green dot appears on the right side of the node.

2
Drag from the green dot

Hold the left mouse button and drag towards the target node.

3
Release over the target node

A popup lets you choose the relation type. Click the desired type.

💡
You can change the relation type afterwards by selecting the edge and using the right-hand side panel.

Deleting a connection

Click the connection line in map view to select it, then press Del or use the delete button in the side panel.

Table view

Structured editing of objects and parameters

Table view shows all nodes in a category as rows, with parameters as columns. Ideal for entering and editing data quickly without navigating the map.

Editing in the table

  • Click a cell to edit its value inline
  • Press Tab to move to the next cell
  • Press Enter to confirm and move to the row below
  • Click the ⋯ button on a row to open the node in the side panel

Column management

Click the "Columns" button per category block to:

  • Hide or show individual columns
  • Reorder columns with the ↑↓ arrows
  • Enable dependency columns (shows connections)

Dependency columns

For each relation type you can enable a dedicated column showing which nodes this node is connected to. Useful for seeing e.g. "which IT systems support capability X?" directly in the table.

Export and import via CSV

Export

Click the CSV button per category. The file is exported with a UTF-8 BOM so it opens correctly in Excel. You can also export as JSON.

Import

Click Import CSV in the toolbar. A dialog lets you map CSV columns to parameters, and you can choose to add to an existing category or create a new one.

💡
CSV import is flexible: column names in the file do not need to match exactly — you map them manually in the import dialog.

Matrix view

Visualise connections between two categories as a grid

Matrix view shows relations between nodes in two different categories as an interactive grid — one category as rows, one as columns. Coloured dots in the cells indicate which type of connection exists.

Example: Capabilities × IT systems

Reporting

Coloured dots correspond to relation types. Dark blue = Hierarchy, Green = Measures, Blue = Supports, Orange = Delivers to.

Clicking in the matrix

  • Click an empty cell to add a connection between the two nodes
  • Click a dot in a cell to remove that specific connection
  • Choose the relation type in the popup that appears

Typical use cases

  • Which IT systems support which capabilities?
  • Which initiatives contribute to which goals?
  • Which KPIs measure which strategic goals?

Perspectives

Filter the map for different audiences

A perspective is a named filter that shows only a selected set of categories. Use perspectives to create tailored views for different audiences — the board sees only goals and KPIs, the CIO sees capabilities and systems, and so on.

Creating a perspective

1
Click "Perspectives"

In the top bar at the top of the app.

2
Click "+ New perspective"

Give the perspective a name and a description.

3
Select categories

Tick which categories should be visible in this perspective.

Example perspectives

  • Board view – Strategic goals + KPIs
  • Portfolio view – Capabilities + Initiatives
  • Technology map – Capabilities + IT systems
  • Operational – Processes + Systems + Teams
✅
"All" always shows every category and is the default active perspective.

Searching the map

Press / or Ctrl+K to activate search. Type a search term to:

  • Dim non-matching nodes in map view
  • Filter rows in table view
  • The search matches against node titles and parameter values

Press Esc to clear the search.

Import and Export

Backup, sharing and integration

Full backup – JSON

Export

Click "Export JSON" in the toolbar. The file contains the complete map including categories, parameters, nodes, connections and perspectives.

Import

Click "Import JSON" and select a previously exported JSON file to restore or share a map.

⚠️
Import overwrites existing data. Take an export backup before importing.

Export to image – SVG and PNG

In map view, click the export button in the toolbar and choose a format:

  • SVG – Vector format, scalable without quality loss. Suitable for presentations and print.
  • PNG – Raster image, easy to paste into documents and emails.

Per-category export and import

JSON export per category is done from the table view export menu.

CSV export creates a file with a UTF-8 BOM that opens correctly in Excel. All parameters and dependency columns are included.

CSV import via the button in the table view toolbar. You can:

  • Map CSV columns to existing parameters
  • Add to an existing category
  • Create a new category from the CSV

Undo and Redo

All changes can be undone up to 20 steps back:

  • Ctrl+Z – Undo last action
  • Ctrl+Shift+Z – Redo (after undo)
  • Also use the undo/redo buttons in the toolbar

Relation types

Customise the four relations:

  • Choose the line colour
  • Choose the dash pattern (solid, dashed, dotted)

Adjust the map's appearance and general preferences:

  • Corner radius – rounding on node cards
  • Minimum width – smallest width for nodes in the map

Tips and tricks

Get up to speed faster

Recommended setup for a new organisation

1
Start with categories

Define the strategic layers you want to map: e.g. Goal → KPI → Capability → Initiative → IT system.

2
Define the parameters

For each category, add the fields you need. Think about what you want to report on — status, owner, deadline, cost?

3
Enter data in table view

Table view is often faster than map view for bulk entry of nodes and values.

4
Draw connections in map view

Switch to map view and drag relations between nodes to build the dependency map.

5
Create perspectives

Build perspectives for different audiences and share the map as PNG or JSON.

Use auto-layout

After entering many nodes and connections the map can become cluttered. Click Auto-layout in the toolbar to tidy up automatically. The result can be adjusted manually afterwards.

The matrix as a tool

Use matrix view to quickly spot "white spots" — missing connections between categories. An empty row or column in the matrix may mean an IT system supports no capability.

✅
Remember to back up! Export a JSON backup regularly, especially after major changes. Save the file somewhere safe — it is your only copy outside the browser.

Timeline / Gantt view

Visualise nodes on a time axis

The timeline view shows nodes as horizontal bars on a calendar-based timeline — just like a Gantt chart. It is useful for initiatives, projects, milestones and other nodes that have date parameters.

Prerequisite

The category you want to display must have at least one parameter of type Date. Without date parameters the timeline will show a warning.

💡
Add parameters "Start date" and "End date" (both of type Date) to categories such as Initiative or Project to get full Gantt bars with duration.

How to use the timeline

  1. Click "Timeline" in the top bar
  2. Select which category should appear as rows
  3. Select the start date parameter (e.g. "Start date")
  4. Select the end date parameter (e.g. "End date") — optional; without it a point marker is shown
  5. Click a row or bar to open the node in the side panel

What is shown

ElementDescription
Coloured barThe node's duration from start date to end date
Red line"Today" marker for quick status assessment
Month headersAutomatic month divisions along the x-axis
Left columnNode name — click to open in the side panel
📌
The timeline updates automatically when you change date values in table or map view. The timeline selection (category and parameters) is saved in the browser.

Group boxes

Visual containers and swim lanes in map view

Group boxes are visual rectangles you draw over the map to gather related nodes. They are purely overlay elements — they do not affect data, calculations or connections, but make the map much easier to read for others.

Creating a group

  1. Go to Map view
  2. Click "⬜ Group" in the toolbar
  3. A new box is placed in the centre of the current visible area
  4. Drag the box by its label and resize with the corner handle

Editing a group

Click the box to select it — the side panel opens with:

  • Name — type directly in the field; it updates immediately
  • Colour — choose from the 12 palette colours
  • Delete group — removes the box (can be undone with Ctrl+Z)
💡
Use group boxes as swim lanes: one box per business domain, one per phase, or one per team. Drag nodes into the boxes to organise the map visually.

Conditional formatting

Automatically colour rows and nodes based on values

Conditional formatting lets you set up rules that automatically colour rows in the table and nodes in the map based on parameter values. A rule such as "If status is red → yellow row colour" makes critical items stand out immediately.

Creating rules

  1. Go to Table view
  2. Click the "Formatting" button next to "Columns" for the desired category
  3. Click "+ Add rule"
  4. Select parameter, operator and value
  5. Select row colour, text colour and optionally bold text
  6. Click "Save"

Available operators

OperatorUsed forExample
equalsExact matchStatus equals red
does not equalAll other valuesStatus does not equal green
is greater thanNumbers above a thresholdCost > 1000000
is less thanNumbers below a thresholdScore < 50
>= / <=Numbers including the thresholdProgress >= 100
containsSubstring searchName contains risk
is emptyMissing valueOwner is empty
is not emptyValue existsStart date is not empty
💡
For status parameters the values red, yellow and green are used in rules (not "Red"/"Yellow"/"Green").

Where do the rules apply?

  • Table view — the entire row is coloured with the chosen row colour and text colour
  • Map view — matching nodes get a coloured top stripe and an adjusted title style

Examples of useful rules

RuleStripe colourEffect
Status equals redRedCritical items stand out
Status equals yellowYellowItems needing follow-up
Progress < 30RedProjects falling behind
Owner is emptyOrangeItems without an owner
Priority equals HighBlueHigh-priority items highlighted
📌
Rules are evaluated in the order they were created. If multiple rules match, styles are merged — the last rule wins in case of conflict.

Dashboard

The dashboard view lets you assemble a custom control panel with widgets from different categories and objects — ideal for management meetings, status reviews and digital screen displays.

Getting started

Click "Dashboard" in the top bar to open the view. The dashboard is empty on first load. Click "+ Add widget" or activate "✎ Edit" mode and choose "+ Widget" to get started.

Widget types

KPI / single value
Shows one parameter value from one object in large, prominent text. Perfect for key figures such as "Customer satisfaction: 87%" or a status indicator. You can choose a target parameter to display a progress bar below the value.
Object count
Large counter showing the number of nodes in a category. Useful for "12 initiatives", "5 IT systems" etc.
Status distribution
Colour-segmented bar (green / yellow / red) showing the distribution of traffic-light statuses in a category. Gives an instant snapshot of portfolio health.
Progress indicator
Progress bar for a numeric or calculated parameter. Coloured automatically: green ≥ 75%, yellow ≥ 40%, red below 40%. You set the maximum value (default 100).
Object list
Mini-table with all nodes in a category and optional columns. Conditional formatting from the category's rules is active in the table.
Text / heading
Free-text box for headings, descriptions or other contextual information that organises the dashboard.
Exception list
Automatically shows all objects in a category that trigger conditional formatting rules — a quick overview of what needs attention.

Grid and positioning

The dashboard uses an invisible 12-column grid with rows of 100 px each. Each widget is placed using four values:

ValueDescriptionExample
ColumnStarting column (1–12)1 = far left
RowStarting row (1–N)1 = top
WidthNumber of columns the widget spans (1–12)4 = one third of the width
HeightNumber of rows (1–10)2 = 200 px tall

In edit mode you can drag and drop widgets to new positions by holding and dragging the widget header.

Editing and configuring

  1. Click "✎ Edit" to activate edit mode
  2. Click "+ Widget" to add a new widget
  3. Select the widget type and configure the data source, position, width and height in the dialog
  4. Click the gear icon (⚙) on a widget to modify it
  5. Click ✕ on a widget to delete it
  6. Click "Done" to exit edit mode
Tip: Combine the status distribution widget, KPI cards and exception list for an effective management view. Use text widgets as headings to divide the dashboard into sections.

Export

Dashboard export is available in the File ▾ menu in the top bar:

  • Dashboard as HTML — Standalone HTML file with all widgets, suitable for distribution or archiving. Includes a "Print / Save as PDF" button.
  • Dashboard as PNG — Opens the dashboard in a new window ready for printing. Use the browser's print-to-PDF feature to save as an image.
Note: Dashboard data is saved in the browser (localStorage) as part of the map. Use File ▾ → JSON (backup) to take a backup that includes the dashboard layout.
